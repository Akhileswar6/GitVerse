Git Log

The git log command is used to view the commit history in a git Repository. It displays a list of commits made to the project.
It shows the commit hash, author, date, and commit message for each commit. This command is useful for tracking changes made to the codebase over time and understanding the history of the project.
To view the history of commits for a repository, you can use the git log command.

Syntax

git log 

  $ git log
  commit d13c7b53d8ff2a2acd967352549337cb359416f0 (HEAD -> master)
  Author: Akhileswar6 <akhilkamale@gmail.com>
  Date:   Sun Apr 20 22:58:29 2025 +0530
      
      Initial commit
      
  commit 11ad9756a3522ee5cf6c2c0370b54ae8c3709cfb
  Author: Akhileswar6 <akhilkamale@gmail.com>
  Date:   Sun Apr 20 00:09:00 2025 +0530
      
      Initial commit

Common Options

  • git log --oneline - Shows each commit in one line (short hash + message).
  • git log --graph - Displays a graphical representation of the commit history.
  • git log --stat - Shows files changed in each commit with number of insertions/deletions.
  • git log -p - Displays the patch (diff) introduced in each commit.
  • git log --author="name" - Filters commits by a specific author.
  • git log --since="date" - Shows commits since a specific date.
  • git log --until="date" - Shows commits until a specific date.
  • git log <filename> - Shows the commit history of a specific file.
