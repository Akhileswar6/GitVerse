Git Status

Git Status

The git status command is used to display the state of the working directory and the staging area. It lets you see which changes have been staged, which haven't, and which files aren't being tracked by Git.

Files in your Git repository folder can be in one of 2 states

  • Tracked - files that Git knows about and are added to the repository.
  • Untracked - files that Git does not know about and are not added to the repository.

Syntax

git status

This will show you a summary of the current state of your repository, including:

  • Current branch – It shows which branch you are on.
  • Changes not staged for commit – Files that have been modified but not yet added to the staging area using git add.
  • Changes to be committed – Files that have been added to the staging area and are ready to be committed.
  • Untracked files – Files that are not being tracked by Git and have not been added to the staging area.
  • Advice/help – Suggestions like running git add or git restore to fix things.

Example

  git status
  On branch master
  Changes not staged for commit:
    (use "git add <file>..." to update what will be committed)
    (use "git restore <file>..." to discard changes in working directory)
          modified:   index.html
          
  Untracked files:
    (use "git add <file>..." to include in what will be committed)
          example.py
          script.js
          style.css
          
    no changes added to commit (use "git add" and/or "git commit -a")


Git Status --short

The git status --short (or git status -s) command provides a compact, two-column summary of your working directory and staging area.

Syntax

git status --short

Common Codes

  • ?? - Untracked files are files in your working directory that haven't been added.
  • A - Newly added files that are staged and ready to be committed.
  •  M - The file has been modified in the working directory but has not been staged yet.
  • M - The file has been modified and the changes have been staged.
  • MM - The first M indicates a change that has been staged (added using git add).
               The second M shows that the file was modified again after staging..
  • D - The file has been deleted in the working directory but not staged for commit .
  • R - The file has been renamed in the working directory, and the change has been staged for commit.
  • C - The file has been copied to a new location and the change has been staged for commit.
  • AM - The file has been both added (newly created) and modified, and the changes have been staged for commit.
  • AD - The file has been added in one branch and deleted in the other branch during a merge conflict.
  • UU - A merge conflict occurred because both branches modified the same part of the same file.
  • !! - The file is ignored by Git, typically due to being listed in a .gitignore file.
