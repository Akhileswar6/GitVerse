Git Init

Creating Git Folder

Now, let's create a new folder for our project.

  • mkdir makes a new directory.
mkdir myproject
  • cd changes the current working directory.
cd myproject

Now that we are in the correct directory. We can start by initializing Git!

  • If you already have a folder/directory you would like to use for Git - Navigate to it in command line, and Initialize a new git repository.


Git Init

The git init command is used to create a new Git repository. When you run git init, Git sets up the necessary configuration files and directories in your project, allowing you to track changes in your files and manage version control. This is the first step in turning a project into a Git repository.

Example

  • git init command is used to Initialize a new git repository in the current directory.
git init
  • git init <directory> command is used to Initialize a new git repository in the specified directory.
git init myproject

Result

Initialized empty Git repository in C:/Users/admin/myproject/.git/



What Happens After You Run git init?

  • Git now knows that it should watch this folder and track changes to the files inside it.
  • Git creates a hidden folder called .git inside your project. This folder contains all the configuration, history, and data Git needs to manage your version control.
  • You won’t see this folder unless you enable hidden files in your file explorer or use ls -a in the terminal.
