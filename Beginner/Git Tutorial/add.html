Git Adding New Files

You just created your first local Git repo, But it is empty.So add some files using any text editor.
Then check the status of the repo using the command git status.

  • For this example, I'm using a basic html program.

<!DOCTYPE html>
<html lang="en">
<head>
   <title>Basic HTML Program</title>
</head>
<body>
   <h1>Hello, World!</h1>
   <p>This is git add command.</p>
</body>
</html>

  • Now save the file as index.html.
  • Now check the status of the repo using the command git status.

  • You will see the following output

$ git status
On branch master

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)
       index.html

nothing added to commit but untracked files present (use "git add" to track)



Git Add

The git add command is used in Git to add changes in your working directory to the staging area. The staging area is a place where Git keeps track of changes that are ready to be committed. This command allows you to select which changes should be included in your next commit.

  • Now you can add the file to the staging area using the command git add <file>.
  • In this case, you can use git add index.html to add the file to the staging area.

Syntax

git add <filename>

Common Usage

  • Add a single file - Adding a specific file to staging area.
git add example.py
  • Add a multiple files - Adding multiple files to staging area.
git add example.py sample.txt
  • Add all changes in the current directory and subdirectories - The . (dot) represents the current directory, and using this command will add all changed files, new files, and deleted files in the directory to the staging area.
git add .
  • Add all changes in the repository - This will add all changes (including deletions) across the entire repository to the staging area.
git add -A

Note :git add doesnâ€™t actually commit the changes, it just stages them for the next commit.
If you make further changes to a file after staging it with git add, you need to run git add again to stage the new changes.
